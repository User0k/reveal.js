Regular expressions

Long, often used to provide a variable as a pattern 
regexp = new RegExp("pattern", "flags")
Short, used in most cases when we know what the pattern is 
regexp = /pattern/gi
i
With this flag the search is case-insensitive
g
Allows to search for all matches
m
Multiline mode
s
Enables 'dotall' mode, that allows a . to match newline character \n
u
Enables full Unicode support
y
Searches at the exact position in the text
Sets can be extremely useful when you try to find the words that differ with 1 character
Beat the heat 'Beat the heat'.match(/[bh]eat/gi) //["Beat", "heat"]
Square brackets may also contain character ranges 
						
	'Lorem ipsum dolor sit amet consectetur adipisicing elit'
		.replace(/[a-zA-Z]/g, '')

	//'       '
Character classes are shorthands for certain character sets
  • \d === [0-9]
  • \w === [a-zA-Z0-9_]
  • \s === [\t\n\v\f\r] (a space symbol)
  • \D !== \d
  • \W !== \w
  • \S !== \s
  • . is a special character class that means 'any character except a newline'
A quantifier is appended to a character or a character class and specifies how many we need
Quantifiers 
						
							'1 12 123 1234'.match(/\d{3,5}/)
							//["123", "1234"]
							'1 12 123 1234'.match(/\d+/g)
							//["1", "12", "123", "1234"]
